快速開始

在 5 分鐘內獲得您的第一個風險評分。您需要從您的 SecureStar 儀表板獲取 API 金鑰。

安裝 SDK

使用 pip 安裝 Python SDK。Node.js 和 Java SDK 也可用。

bash
pip install securestar-sdk

配置您的 API 金鑰

將您的 API 金鑰設置為環境變量或直接傳遞給客戶端。

python
import securestar

client = securestar.Client(
  api_key="ss_live_..."
)

評分您的第一筆交易

使用交易數據調用 score() 方法。響應包括風險評分、決策和解釋。

python
result = client.score({
  "transaction_id": "txn_9x2q8a",
  "amount": 4820.00,
  "currency": "USD",
  "account_id": "acct_h82jk",
  "merchant_id": "merch_4x9m",
  "ip_address": "104.22.1.84",
  "device_id": "dev_8k2l",
})

print(result.risk_score) # 0.08
print(result.decision) # "approve"
print(result.latency_ms) # 43

POST /v2/score

實時評分單筆交易。返回風險評分、決策建議、解釋和審計 ID。

POST https://api.securestar.io/v2/score 實時交易評分

請求參數

參數類型必需描述
transaction_id string 必需 您的唯一交易標識符
amount number 必需 指定貨幣的交易金額
currency string 必需 ISO 4217 貨幣代碼（例如 USD）
account_id string 必需 您的內部帳戶或用戶標識符
merchant_id string 可選 商戶風險信號的商戶標識符
ip_address string 可選 用於地理和代理信號的客戶端 IP 地址
device_id string 可選 用於新設備信號的設備指紋
metadata object 可選 包含在審計日誌中的任意鍵值對
💡 延遲提示：傳遞所有可用的可選字段。每個額外的信號都能減少模型的不確定性，並能提高高達 8% 的準確性。額外字段的延遲開銷低於 2ms。

Kafka 連接器

SecureStar Kafka 連接器訂閱您的交易主題並將評分事件發布到結果主題 — 無需修改您現有的生產者。

⚡ 延遲：Kafka 連接器在 Kafka 消費者到消費者路徑中增加了 8ms 的中位數。由於消除了 HTTP 開銷，這通常比 REST API 更快。
yaml — securestar-kafka.yaml
connector:
  input_topic: payments.transactions.raw
  output_topic: payments.transactions.scored
  error_topic: payments.transactions.errors
  consumer_group: securestar-scorer
  parallelism: 8

mapping:
  transaction_id: $.id
  amount: $.amount_usd
  account_id: $.user_id

performance:
  timeout_ms: 150
  fallback_score: 0.5
  fallback_decision: flag

審計日誌

每個評分決策都會自動寫入不可變的、加密簽名的審計日誌。日誌可通過 API 查詢，導出到您的 SIEM，並默認保留 7 年。

GET /v2/audit/events 查詢審計事件日誌
參數類型描述
fromdate日期範圍的開始（ISO 8601）
todate日期範圍的結束（ISO 8601）
decisionstring按 approve、flag 或 decline 過濾
account_idstring按帳戶過濾
rule_hitstring過濾觸發特定規則的事件
