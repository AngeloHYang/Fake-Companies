關於 SecureStar

我們構建銀行能夠真正解釋的欺詐基礎設施

SecureStar 由工程師創立，他們在金融科技和機器學習平台內工作多年，目睹同樣的問題重複出現：面臨真正欺詐風險的公司和擁有真正解決方案的供應商無法找到彼此，或者即使找到了也無法互相信任。

我們的使命

快速、誠實且可審計的欺詐檢測

大多數欺詐平台由數據科學家構建，他們以模型性能指標為思考方式。SecureStar 由工程師構建，他們也曾坐在監管機構面前解釋為何拒絕某筆交易。

這種經驗影響了我們所做的每一個產品決策。速度很重要。 一個增加200毫秒授權時間的欺詐檢查是用戶能夠感受到的。可解釋性很重要。 一個無法向合規官解釋其決策的模型是一種負擔，而不是資產。控制很重要。 一刀切的欺詐模型不適合任何人。

我們構建了 SecureStar，成為我們希望在桌子另一邊時存在的平台。

我們的故事

從痛點到平台

💡
2019 — 問題
意識到
聯合創始人 Priya Nair 和 James Kowalski 在不同的金融科技公司工作。兩人都經歷了同樣的挫折：擁有令人印象深刻的推銷簡報的欺詐供應商在需要向合規檢查員解釋決策時崩潰，或者無法達到低於100毫秒的延遲預算。
🛠️
2020 — 創立
SecureStar Labs 成立
Priya 和 James 在舊金山成立 SecureStar Labs。由 Emergence Capital 領投的480萬美元種子輪融資。首批8名工程師來自 Stripe、Palantir 和 Two Sigma。核心使命：構建同時滿足工程師和合規團隊的欺詐平台。
🚀
2021 — 產品
v1 推出混合引擎和可解釋性
首個生產版本推出，具有混合機器學習和規則引擎。早期客戶 Nexo Payments 與其現有供應商進行平行測試 — SecureStar 提供31%更少的誤報和在其 Kafka 堆棧上的低於60毫秒的 p99 延遲。A輪融資達1800萬美元。
📈
2022 — 擴展
Kafka 原生連接器和 Python SDK 發布
工程團隊發布了 Kafka 連接器和 Python SDK，將典型的集成時間從3-4週縮短到不到2週。客戶數量達到40。首次企業 AML 合規部署，具有完整的監管導出管道。
🔬
2023 — 研究
自定義模型調整 API 和邊緣部署測試版
基於客戶數據的自定義模型微調推出，消除了“一刀切”問題。邊緣部署測試版為選定的企業客戶實現了低於20毫秒的 p50。完成 SOC 2 類型 II 認證。
🌍
2024–2025 — 擴展
B輪融資和國際擴展
B輪融資達4200萬美元。推出符合 GDPR 的歐盟數據駐留。團隊增長至約50人。平台處理超過30億筆交易。新功能：動態規則更新、A/B 影子模式和美國客戶的 FinCEN SAR 導出。

我們的信念

⚡

速度是一種功能

延遲不是技術附註。在支付授權中，每毫秒都是一個用戶體驗決策。我們將延遲服務水平協議視為產品承諾，而不是願景。

🔍

不透明是一種負擔

無法自我解釋的人工智慧系統不值得信賴 — 它是一種監管風險。我們相信每個自動化的金融決策都應該能夠向受影響的人解釋。

🎛️

控制權屬於客戶

您比我們更了解您的客戶。您的合規團隊應該擁有規則。您的數據科學家應該調整模型。我們提供基礎設施；您做出決策。

🤝

集成應該是無聊的

最好的供應商集成應該是幾天而不是幾個月，且不需要遷移項目。我們對 SDK 的易用性、連接器的簡單性和零重工部署充滿熱情。
