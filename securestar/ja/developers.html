クイックスタート

5分以内に最初のリスクスコアを取得します。SecureStarダッシュボードからAPIキーが必要です。

SDKをインストール

pipを使用してPython SDKをインストールします。Node.jsとJavaのSDKも利用可能です。

bash
pip install securestar-sdk

APIキーを設定

APIキーを環境変数として設定するか、クライアントに直接渡します。

python
import securestar

client = securestar.Client(
  api_key="ss_live_..."
)

最初のトランザクションをスコアリング

トランザクションデータを使用してscore()メソッドを呼び出します。レスポンスにはリスクスコア、決定、説明が含まれます。

python
result = client.score({
  "transaction_id": "txn_9x2q8a",
  "amount": 4820.00,
  "currency": "USD",
  "account_id": "acct_h82jk",
  "merchant_id": "merch_4x9m",
  "ip_address": "104.22.1.84",
  "device_id": "dev_8k2l",
})

print(result.risk_score) # 0.08
print(result.decision) # "approve"
print(result.latency_ms) # 43

POST /v2/score

リアルタイムで単一のトランザクションをスコアリングします。リスクスコア、決定推奨、説明、監査IDを返します。

POST https://api.securestar.io/v2/score リアルタイムトランザクションスコアリング

リクエストパラメータ

パラメータタイプ必須説明
transaction_id string 必須 あなたのユニークなトランザクション識別子
amount number 必須 指定された通貨でのトランザクション金額
currency string 必須 ISO 4217通貨コード（例：USD）
account_id string 必須 あなたの内部アカウントまたはユーザー識別子
merchant_id string オプション マーチャントリスクシグナルのためのマーチャント識別子
ip_address string オプション ジオおよびプロキシシグナルのためのクライアントIPアドレス
device_id string オプション 新しいデバイスシグナルのためのデバイスフィンガープリント
metadata object オプション 監査ログに含まれる任意のキーと値のペア
💡 レイテンシーのヒント: 利用可能なすべてのオプションフィールドを渡します。追加のシグナルはモデルの不確実性を減らし、精度を最大8%向上させることができます。追加フィールドのレイテンシーオーバーヘッドは2ms未満です。

Kafka コネクタ

SecureStar Kafkaコネクタは、トランザクショントピックにサブスクライブし、スコアリングされたイベントを結果トピックに公開します。既存のプロデューサーを変更することなく利用できます。

⚡ レイテンシー: Kafkaコネクタは、Kafkaのコンシューマーからコンシューマーへのパスに8msの中央値を追加します。これは、HTTPオーバーヘッドが排除されるため、通常REST APIよりも高速です。
yaml — securestar-kafka.yaml
connector:
  input_topic: payments.transactions.raw
  output_topic: payments.transactions.scored
  error_topic: payments.transactions.errors
  consumer_group: securestar-scorer
  parallelism: 8

mapping:
  transaction_id: $.id
  amount: $.amount_usd
  account_id: $.user_id

performance:
  timeout_ms: 150
  fallback_score: 0.5
  fallback_decision: flag

監査ログ

すべてのスコアリング決定は、自動的に不変で暗号的に署名された監査ログに書き込まれます。ログはAPIを介してクエリ可能で、SIEMにエクスポート可能で、デフォルトで7年間保持されます。

GET /v2/audit/events 監査イベントログをクエリ
パラメータタイプ説明
fromdate日付範囲の開始（ISO 8601）
todate日付範囲の終了（ISO 8601）
decisionstringapprove、flag、またはdeclineでフィルタリング
account_idstringアカウントでフィルタリング
rule_hitstring特定のルールがトリガーされたイベントをフィルタリング
