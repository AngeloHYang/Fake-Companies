SecureStar 플랫폼

SecureStar를 현대 금융 시스템의 사기 탐지 인프라로 만드는 다섯 가지 기둥을 깊이 있게 살펴봅니다.

기둥 1

초저지연 추론

사기는 멈추지 않습니다. 탐지 계층도 멈추지 않아야 합니다. SecureStar의 추론 파이프라인은 하나의 제약 조건을 중심으로 설계되었습니다: 트래픽 급증 시에도 99번째 백분위수에서 80ms 이하를 유지합니다.

  • 최적화된 ONNX 추론 런타임 — 모델은 직렬화 오버헤드 없이 제공됩니다
  • 엣지 배포 옵션으로 스코어링 로직을 데이터 소스에 더 가깝게 배치
  • Kafka 소비자 그룹을 통한 스트리밍 우선 설계로 10ms 이하의 이벤트 픽업
  • 수평 자동 확장으로 10배 트래픽 급증 시 SLA 유지
  • 기업 테넌트별 전용 인프라 — 소음이 없는 이웃 효과 없음

기둥 2

설명 가능한 AI — 모든 결정, 매번

대부분의 ML 사기 벤더는 점수를 제공합니다. SecureStar는 이유를 제공합니다. 모든 위험 평가에는 기능 속성, 규칙 속성 및 실제로 사용할 수 있는 인간이 읽을 수 있는 설명이 포함됩니다.

  • SHAP 기반 기능 속성은 각 점수를 주도하는 주요 요소를 식별합니다
  • 규칙 엔진 결정은 코드가 아닌 평문으로 표시됩니다
  • 전체 모델 설명 가능성은 AML Directive 6 및 FinCEN 감사 요구 사항을 충족합니다
  • 설명 페이로드는 지연 프리미엄 없이 모든 API 응답에 포함됩니다
  • 사후 사건 포렌식 분석을 위한 검색 가능한 설명 기록

기둥 3

하이브리드 결정 엔진

순수 ML은 준수를 위해 너무 불투명합니다. 순수 규칙은 사기에 대해 너무 취약합니다. SecureStar는 두 가지를 병렬로 실행하고 이를 단일 결정으로 조정하여 동시에 제어와 지능을 제공합니다.

  • 규칙 엔진 레이어: 준수 팀이 코드 없는 인터페이스에서 하드 규칙 정의
  • ML 모델 레이어: 그라디언트 부스팅 앙상블이 규칙이 놓치는 새로운 패턴을 포착
  • 결정 조정: 규칙 결과와 모델 점수 간의 가중치 조정 가능
  • 규칙 재정의: 준수는 ML 점수와 상관없이 승인/거부 강제 가능
  • 완전한 투명성: 두 레이어의 출력이 모든 API 응답에 포함됨

기둥 4

맞춤형 모델 튜닝

모든 것을 아우르는 사기 모델은 작동하지 않습니다. SecureStar는 자체 거래 데이터로 학습하고, 위험 수용도에 맞게 정밀도/재현율 임계값을 조정하며, 엔지니어링 팀을 기다리지 않고 동적 규칙 업데이트를 푸시할 수 있게 합니다.

  • API 또는 UI를 통해 자체 레이블이 지정된 과거 거래 데이터로 미세 조정
  • 조정 가능한 결정 임계값: 거짓 긍정과 거짓 부정 간의 균형 조정
  • 동적 규칙 업데이트는 60초 이내에 프로덕션에 푸시됨
  • A/B 섀도 모드: 전환 전에 라이브 트래픽에서 모델 변경 테스트
  • 모델 성능 대시보드는 드리프트, 정밀도, 재현율 및 F1을 실시간으로 표시
# 파이썬 SDK — 임계값 튜닝

from securestar import ModelConfig

config = ModelConfig(
  base_model="ss-fintech-v3",
  training_data="s3://your-bucket/txns/",
  label_column="is_fraud",
  thresholds={
    "approve": 0.30,
    "flag":   0.65,
    "decline": 0.88,
  }
)

job = config.train()  # 비동기
job.wait()
print(job.metrics)
# 정밀도: 0.943  재현율: 0.917
기둥 5

준수 우선 아키텍처

SecureStar는 규제 금융 기관의 준수 및 법무 팀과 긴밀히 협력하여 설계되었습니다. 준수는 추가 기능이 아니라 데이터 모델에 내장되어 있습니다.

  • 변경 불가능한 암호화 서명된 감사 로그는 기본적으로 7년 동안 저장됩니다
  • SOC 2 Type II 인증; PCI-DSS 레벨 1 준수 인프라
  • FinCEN SAR, FATF, EBA 형식의 원클릭 규제 내보내기
  • AML 규칙 라이브러리: 구조화, 스머핑, 레이어링 패턴에 대한 사전 구축된 규칙
  • KYC 통합 훅: 원하는 신원 확인 제공업체를 연결
  • GDPR / CCPA 데이터 거주지 제어: EU 또는 미국 저장 지역 선택
