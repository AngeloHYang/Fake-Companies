빠른 시작

5분 이내에 첫 번째 위험 점수를 받으세요. SecureStar 대시보드에서 API 키가 필요합니다.

SDK 설치

pip을 사용하여 Python SDK를 설치하세요. Node.js 및 Java SDK도 사용할 수 있습니다.

bash
pip install securestar-sdk

API 키 구성

API 키를 환경 변수로 설정하거나 클라이언트에 직접 전달하세요.

python
import securestar

client = securestar.Client(
  api_key="ss_live_..."
)

첫 번째 거래 점수 매기기

거래 데이터를 사용하여 score() 메서드를 호출하세요. 응답에는 위험 점수, 결정, 설명이 포함됩니다.

python
result = client.score({
  "transaction_id": "txn_9x2q8a",
  "amount": 4820.00,
  "currency": "USD",
  "account_id": "acct_h82jk",
  "merchant_id": "merch_4x9m",
  "ip_address": "104.22.1.84",
  "device_id": "dev_8k2l",
})

print(result.risk_score) # 0.08
print(result.decision) # "approve"
print(result.latency_ms) # 43

POST /v2/score

실시간으로 단일 거래를 점수화합니다. 위험 점수, 결정 추천, 설명 및 감사 ID를 반환합니다.

POST https://api.securestar.io/v2/score 실시간 거래 점수화

요청 매개변수

매개변수유형필수설명
transaction_id 문자열 필수 고유 거래 식별자
amount 숫자 필수 지정된 통화의 거래 금액
currency 문자열 필수 ISO 4217 통화 코드 (예: USD)
account_id 문자열 필수 내부 계정 또는 사용자 식별자
merchant_id 문자열 선택 상인 위험 신호를 위한 상인 식별자
ip_address 문자열 선택 지리 및 프록시 신호를 위한 클라이언트 IP 주소
device_id 문자열 선택 새로운 장치 신호를 위한 장치 지문
metadata 객체 선택 감사 로그에 포함된 임의의 키-값 쌍
💡 지연 시간 팁: 사용 가능한 모든 선택적 필드를 전달하세요. 각 추가 신호는 모델의 불확실성을 줄이고 정확성을 최대 8%까지 향상시킬 수 있습니다. 추가 필드의 지연 시간 오버헤드는 2ms 미만입니다.

Kafka 커넥터

SecureStar Kafka 커넥터는 거래 주제를 구독하고 점수화된 이벤트를 결과 주제로 게시합니다 — 기존 프로듀서를 수정하지 않고도 가능합니다.

⚡ 지연 시간: Kafka 커넥터는 Kafka 소비자-소비자 경로에 평균 8ms를 추가합니다. 이는 HTTP 오버헤드가 제거되어 REST API보다 일반적으로 빠릅니다.
yaml — securestar-kafka.yaml
connector:
  input_topic: payments.transactions.raw
  output_topic: payments.transactions.scored
  error_topic: payments.transactions.errors
  consumer_group: securestar-scorer
  parallelism: 8

mapping:
  transaction_id: $.id
  amount: $.amount_usd
  account_id: $.user_id

performance:
  timeout_ms: 150
  fallback_score: 0.5
  fallback_decision: flag

감사 로그

모든 점수 결정은 자동으로 변경 불가능하고 암호화 서명된 감사 로그에 기록됩니다. 로그는 API를 통해 쿼리할 수 있으며, SIEM으로 내보낼 수 있으며 기본적으로 7년 동안 보관됩니다.

GET /v2/audit/events 감사 이벤트 로그 쿼리
매개변수유형설명
from날짜날짜 범위 시작 (ISO 8601)
to날짜날짜 범위 종료 (ISO 8601)
decision문자열approve, flag, 또는 decline으로 필터링
account_id문자열계정으로 필터링
rule_hit문자열특정 규칙이 트리거된 이벤트 필터링
