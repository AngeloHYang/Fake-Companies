快速开始

在5分钟内获得您的第一个风险评分。您需要从您的SecureStar仪表板获取API密钥。

安装SDK

使用pip安装Python SDK。Node.js和Java SDK也可用。

bash
pip install securestar-sdk

配置您的API密钥

将您的API密钥设置为环境变量或直接传递给客户端。

python
import securestar

client = securestar.Client(
  api_key="ss_live_..."
)

评分您的第一个交易

使用交易数据调用score()方法。响应包括风险评分、决策和解释。

python
result = client.score({
  "transaction_id": "txn_9x2q8a",
  "amount": 4820.00,
  "currency": "USD",
  "account_id": "acct_h82jk",
  "merchant_id": "merch_4x9m",
  "ip_address": "104.22.1.84",
  "device_id": "dev_8k2l",
})

print(result.risk_score) # 0.08
print(result.decision) # "approve"
print(result.latency_ms) # 43

POST /v2/score

实时评分单笔交易。返回风险评分、决策建议、解释和审计ID。

POST https://api.securestar.io/v2/score 实时交易评分

请求参数

参数类型必需描述
transaction_id string 必需 您的唯一交易标识符
amount number 必需 指定货币的交易金额
currency string 必需 ISO 4217货币代码（例如USD）
account_id string 必需 您的内部账户或用户标识符
merchant_id string 可选 商户风险信号的商户标识符
ip_address string 可选 用于地理和代理信号的客户端IP地址
device_id string 可选 用于新设备信号的设备指纹
metadata object 可选 包含在审计日志中的任意键值对
💡 延迟提示：传递所有可用的可选字段。每个额外的信号都能减少模型的不确定性，并能提高高达8%的准确性。额外字段的延迟开销低于2ms。

响应

json
{
  "transaction_id": "txn_9x2q8a",
  "risk_score": 0.08,
  "decision": "approve",
  "confidence": 0.97,
  "latency_ms": 43,
  "explanation": {
    "top_factors": ["known_merchant", "consistent_geo", "low_velocity"],
    "rule_hits": [],
    "feature_attribution": [
      { "feature": "merchant_trust_score", "weight": -0.14 },
      { "feature": "account_age_days", "weight": -0.09 }
    ],
    "human_summary": "来自已知商户的低风险交易，地理位置符合预期",
    "model_version": "v3.2.1"
  },
  "audit_id": "aud_7gk2p1"
}

Kafka连接器

SecureStar Kafka连接器订阅您的交易主题，并将评分事件发布到结果主题——无需修改您现有的生产者。

⚡ 延迟：Kafka连接器在Kafka消费者到消费者路径中增加了8ms的中位数。这通常比REST API更快，因为消除了HTTP开销。
yaml — securestar-kafka.yaml
connector:
  input_topic: payments.transactions.raw
  output_topic: payments.transactions.scored
  error_topic: payments.transactions.errors
  consumer_group: securestar-scorer
  parallelism: 8

mapping:
  transaction_id: $.id
  amount: $.amount_usd
  account_id: $.user_id

performance:
  timeout_ms: 150
  fallback_score: 0.5
  fallback_decision: flag

审计日志

每个评分决策都会自动写入不可变的、加密签名的审计日志。日志可通过API查询，可导出到您的SIEM，并默认保留7年。

GET /v2/audit/events 查询审计事件日志
参数类型描述
fromdate日期范围的开始（ISO 8601）
todate日期范围的结束（ISO 8601）
decisionstring按approve、flag或decline过滤
account_idstring按账户过滤
rule_hitstring过滤触发特定规则的事件
