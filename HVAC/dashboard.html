Home / Agent Dashboard

This is what Mike sees the moment he opens the app — whether he's still in bed at 6:45 AM or getting in the truck. The agent has been working all night, and the dashboard is its morning report. Four status cards at a glance. Three most recent actions. One voice button. No menus to dig through.

Core Design Principles

01
Mobile-first, voice-first
The owner is in a truck or on a roof. The primary input is voice. Big tap targets, minimal typing.
02
The agent is the interface
Not a dashboard with menus. A conversational thread is the home screen. Cards and action buttons appear inline.
03
Two visual modes
Owner ↔ Agent thread (foreground) and Agent ↔ Agent activity (background — a live feed of what the agent is doing).
04
Trust through transparency
When the agent acts autonomously (ordering parts, filing permits), the owner sees it in real-time as a collapsible card.

Key Elements

  • 📊
    Status Cards (top section)
    Four real-time counters: new leads handled, jobs today, parts arriving, permits pending. Tapping any card drills into that category. All four update the moment the agent acts.
  • ⚡
    Recent Agent Activity Feed
    A live feed showing what the agent did while the owner was driving, sleeping, or on-site. Color-coded tags: confirmed (green), pending (orange), in-transit (blue). Each item is tappable to see the full context or intervene.
  • 🎙️
    Persistent Voice Button
    The primary interaction. Always visible at the bottom. Mike holds it and speaks naturally: "What jobs do I have tomorrow?" or "Order me a Carrier 3-ton from Ferguson." The agent responds inline.
  • 🤖
    Agent Status Badge
    The pulsing green "ACTIVE" badge tells the owner the agent is on and available. If connectivity drops or the agent is processing, this badge changes state so Mike knows before he tries to talk.
What's happening while Mike sleeps

The agent handled 3 incoming leads overnight — qualifying each one, capturing equipment info, and booking appointments. It ordered a condenser unit from Grainger for tomorrow's install, and it's waiting on permit approval from the City of Austin. Mike opens the app and sees all of this in 10 seconds. No calls to return. No parts to order. Just get in the truck.
